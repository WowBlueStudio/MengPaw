# MengPaw 功能闭环审计方法论

> 从 #1 插件系统、#2 GitHub 连接两轮深度审计中提炼。
> 适用于任何 MengPaw 子系统：会话管理、记忆孪生、触发器引擎……

---

## 三层十二问

任何功能闭环必须在这三个维度上经得起追问：

### 1. Agent 认知层 — "Agent 是不是这个领域的专家？"

| # | 问题 | 检查要点 |
|---|------|---------|
| 1.1 | Agent 知道这个功能的存在吗？ | 系统提示词有没有提及？有没有 Few-shot 示例？ |
| 1.2 | Agent 知道从哪里获取资源吗？ | 下载源、存储目录、配置入口是否在提示词中明确？ |
| 1.3 | Agent 知道用什么命令操作吗？ | CLI.md / self.tools 是否覆盖？命令返回格式是否 LLM 友好？ |
| 1.4 | Agent 知道功能的简介/描述吗？ | 列表命令是否附带 description？搜索命令是否可发现？ |
| 1.5 | 操作成功后 Agent 得到指引了吗？ | 安装/创建后是否自动返回后续步骤？是否有 Skill 可查阅？ |
| 1.6 | Agent 有合理的权限边界吗？ | 哪些操作不该允许？删除/卸载有无保护？依赖检查？ |

### 2. 软件逻辑层 — "代码和 UI 是否支撑闭环？"

| # | 问题 | 检查要点 |
|---|------|---------|
| 2.1 | 有固定、独立的存储目录吗？ | 目录是否可预测？Agent 能否用文件系统命令访问？ |
| 2.2 | Agent 能否验证文件系统状态？ | 有无 verify/size/stat 命令？能否区分"安装"和"文件丢失"？ |
| 2.3 | UI 是否提供操作反馈？ | 下载动画？进度百分比（真实值）？成功/失败提示？ |
| 2.4 | 操作后是否自动注册/生效？ | install → activate → 命令可用，是否一气呵成？ |
| 2.5 | 提供"彻底删除"和"临时禁用"两种粒度吗？ | 删除清文件 + 禁用只停命令？UI 按钮是否区分？ |
| 2.6 | 异常路径有处理吗？ | 下载失败有重试？网络错误有建议？数据损坏有恢复？ |

### 3. 服务/基础设施层 — "外部依赖是否可靠？"

| # | 问题 | 检查要点 |
|---|------|---------|
| 3.1 | 主要外部依赖有备份吗？ | GitHub 挂了有 Gitee？Gitee 挂了有 ghproxy？单点故障？ |
| 3.2 | 失败时是否告知 Agent 替代方案？ | 错误消息含建议吗（检查网络、使用 VPN、尝试镜像）？ |
| 3.3 | 版本/更新信息是否传递给 Agent？ | 版本号比较 + changelog + 升级路径是否完整？ |
| 3.4 | 网络受限场景是否告知 Agent？ | 系统提示词 + 错误消息 + 工具命令三通道覆盖？ |
| 3.5 | 数据完整性有保障吗？ | SHA256 校验？原子写入（tmp + rename）？损坏自动恢复？ |
| 3.6 | 降级路径是否自动化？ | GeoRouter 自动切换？缓存兜底？无需用户手动干预？ |

## 使用方法

1. **选一个子系统**（如会话管理、记忆孪生、触发器引擎）
2. **逐条过**：对每一条回答"是/否/部分"
3. **标记缺口**：否和部分的项目就是待修复的 Bug 或待实现的功能
4. **P0/P1/P2 分级**：
   - P0 = Agent 无法自主完成操作 / 数据丢失风险 / 用户无感知
   - P1 = 功能可用但体验差 / 缺少便利工具 / 无容错
   - P2 = 锦上添花 / 低频场景 / 可后续迭代
5. **按优先级施工**

## 实战案例

### #1 插件系统审计

| 发现 | 层级 | 严重度 | 修复 |
|------|------|:----:|------|
| `plugin.marketplace` 不返回 description | 1.4 | P0 | 输出加 description |
| 安装后不返回命令摘要 | 1.5 | P0 | 成功消息追加命令列表 + skill 提示 |
| 系统提示词无插件管理 section | 1.1/1.2 | P0 | PromptEngine 新增 section |
| 内置插件可被卸载 | 1.6 | P0 | UNINSTALLABLE 白名单 |
| 卸载不清理 JAR | 2.2 | P0 | PluginManager.uninstall 删文件 |
| 错误消息无建议 | 3.2 | P0 | 附加 self.tools/VPN/Gitee 提示 |
| 下载进度 0% | 2.3 | P1 | 使用 Content-Length |

### #2 GitHub 连接审计

| 发现 | 层级 | 严重度 | 修复 |
|------|------|:----:|------|
| UpdatePlugin 无 fallback | 3.1 | P0 | GitHub→Gitee→ghproxy 三级 |
| net-plugin 无代理能力 | 1.5/3.4 | P0 | 新增 net.proxy 命令 |
| 错误消息不提 VPN | 3.2/3.4 | P0 | curl/post 错误加 GitHub URL 检测 |
| 系统提示词不提 net.proxy | 1.1/3.4 | P0 | 提示词加 net.proxy 入口 |
| ErrorReportPlugin Gitee only | 3.1 | P2 | error.setup 可手动切 |
