# Make Plan — 任务拆解与计划

当任务需要多步拆解、步骤之间有依赖关系、需要明确检查点时，使用本 Skill。

## 核心原则

- **只要计划，不要代执行**：用 `self.tools` 了解自己有什么能力，然后制定计划
- **步骤必须具体**：不能用抽象建议代替可执行步骤
- **包含验证方式**：每步说明完成标志

## 适用场景

- 任务需要多步拆解
- 步骤之间有依赖关系
- 涉及多个命令/模块/系统
- 用户要求先给出计划

## 执行流程

### 1. 分析任务
- 理解用户最终目标
- 用 `self.tools` 确认有哪些可用命令
- 用 `agent.memory` 查询是否有相关经验

### 2. 制定计划
向用户输出：

```
## 执行计划：{任务简述}

| 步骤 | 操作 | 预期结果 | 验证方式 |
|------|------|---------|---------|
| 1 | ... | ... | ... |
| 2 | ... | ... | ... |

预计共 N 步。确认后开始执行。
```

### 3. 逐步执行
- 用户确认后按顺序执行
- 每步执行完汇报结果
- 步骤失败时分析原因，调整计划

### 4. 完成汇报
全部完成后汇总：
- 完成了哪些步骤
- 最终结果
- 是否有遗留问题

## 不适合的场景

- 只需要一个命令就能完成的事
- 缺少关键信息（先问用户）
- 连任务目标都没理解清楚（先澄清）
